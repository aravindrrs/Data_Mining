In [143]:
from scipy.stats import zscore
scaled_college=zscore(college_num)

We are using Zscore scaler to tranform the data set into normalized values which is ideal for clustering because we need to compare similarities between samples based on distances. Hence this has to be done before dissimilar matrix calculation.We are also interested in standardization which accurately represents the maximum variance especially useful for Principal component analysis later. If we use min max scaling, we might supress the variance and may include outliers within our scaling limits. Hence we prefer Zscore normalization for better clustering analysis.

Single vs Complete linkages
The above arrays represent the cluster labels in the first column and the number of samples in the second column.
Comparing both the tables, we can see that single linkage clustering has grouped too many samples (majority) under the label "2" ,
While actually there are 6 other artists and almost they have no songs under them. This clustering is not very effective. The farther elements belonging to the same cluster are closer to the elements of other clusters, hence there is a good chance of these farther elements being misclassfied and mapped to the wrong cluster. This is the reason we cannot properly divide the data and spread them out in other clusters.Hence we can say that, observing the number of elements(songs) clustered under each artists , Complete linkage clustering is effective and reliable than the single linkage.
